# 过程记录 (Walkthrough): 订单分析

## 2026-01-22 (第二阶段)
- **洞察**: 发现“平均值”掩盖了事实真相。切换为分布分析与聚类分析。清理了陈旧脚本。

## 2026-01-22 (第三阶段 - V4 战略级重构)
- **挑战**: 初步的折扣弹性分析显示结果为空。
- **发现**: 识别出“价格固化”现象——大多数单品从未调整过挂牌价；折扣大多源于订单级的满减券分摊。
- **方案**: 重构价格策略模块 (`PricingStrategy`)，引入“折扣类型”穿透逻辑，并增加了“价格审计”模块。
- **方法论**: 采用“统计自适应”技术（利用分位数和标准分 Z-Score）来动态定义指标的“高”与“低”。
- **产出**: 交付了 V4.2 战略诊断报告。
    - **全局**: 包含跨渠道矩阵、价格带对比、全网头部/长尾商品。
    - **渠道**: 包含商品人格矩阵、客单价偏态、时段热力图、购物篮指纹聚类。
- **最终结果**: 生成了具备专业样式和深度业务术语的高保真 HTML 报告。
