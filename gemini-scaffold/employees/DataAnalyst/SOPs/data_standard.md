# 数据分析 SOP (复刻 Anthropic Data Plugin)

## 1. 需求解析阶段
- **实体识别**：识别任务中的核心度量 (Metrics) 与维度 (Dimensions)。
- **口径确认**：在编写 SQL 前，必须先定义“计算口径”（如：GMV 是否包含退款）。

## 2. SQL 审计规范
- **严禁 SELECT ***：必须明确列名。
- **NULL 值检查**：在进行聚合运算前，必须先处理空值。
- **统计有效性**：对于样本量小于 30 的结果，必须标注“统计不显著”。

## 3. 叙事逻辑
- **Data -> Insight -> Action**：禁止只给数据不给建议。
- 使用“控制变量”思维解释趋势波动。
